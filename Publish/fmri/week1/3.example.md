## A Concrete Example

Assume there is one record in the database:

| fMRI Signal (encoded) | Corresponding Text Description |
|-----------------------|--------------------------------|
| [0.23, -0.45, 0.67, ...] | "A person walking a dog in a park" |

Now a new fMRI scan arrives:

### Step 1: Encode the new data
```
New fMRI signal → fMRI encoder → Query vector [0.21, -0.43, 0.65, ...]
```

### Step 2: Compare against the database (FAISS similarity search)
```
Compute similarity:
- Database record 1: 0.98 (very similar)
- Database record 2: 0.52
- Database record 3: 0.31
...

Select Top-K most similar (e.g., K=3)
```

### Step 3: Retrieve the similar texts
```
Retrieved texts:
1. "A person walking a dog in a park" (similarity 0.98)
2. "A person jogging with their pet" (similarity 0.85)
3. "People exercising outdoors" (similarity 0.76)
```

### Step 4: LLM generates text
```
Feed the retrieved texts as "reference evidence" to the LLM:

Prompt:
"Below are text descriptions of what other people saw while their brain activity was recorded:
- Description 1: A person walking a dog in a park
- Description 2: A person jogging with their pet
- Description 3: People exercising outdoors

Based on these references, describe what this person is currently watching:"

LLM Output:
"A person walking with their dog in an outdoor park setting"
```

> **New fMRI comes in → search the database for "whose brain activity looked most similar to this before" → take the text descriptions they saw/thought → feed those as reference context to an LLM → generate the current description.**
